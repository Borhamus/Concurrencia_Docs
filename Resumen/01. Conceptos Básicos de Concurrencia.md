
**Proceso:** Un **Proceso** es un programa en ejecución que puede ser interrumpido (una instancia de un programa)

**PCB (Bloque de Control de Procesos):** **Estructura de datos fundamental** que el SO utiliza para administrar y controlar cada proceso en ejecución.

Sirve como un depósito de almacenamiento donde el sistema **guarda todo el contenido del proceso en el momento de ser interrumpido.** Esto permite que el proceso pueda reanudar su ejecución más tarde exactamente desde el punto donde se detuvo, con toda la información necesaria intacta.

**PCB en Concurrencia**
Gracias a la PCB el sistema puede realizar el intercalado (interleaving) de procesos, alternando rápidamente el procesador entre diferentes tareas sin perder el progreso de ninguna de ellas.

**Estado de los procesos**
-New: El proceso se está creando.
-Ready: El proceso está esperando que se le asigne el procesador.
-Running: El proceso está siendo ejecutado por el CPU.
-Waiting: El proceso espera un evento.
-Terminated: Termina su ejecución.


**Concepto de Clasificación de los Procesos**
**-Independientes:** No afecta ni es afectado por la ejecución de sí mismo y de otros procesos.
**-Cooperantes:** Puede afectar o ser afectado por la ejecución de sí mismo y de otros procesos, generalmente porque comparten información o recursos.
* Comparten información.
* Aceleran el proceso de computo
* Modularidad

// Estos generan problemas de sincronización.

**Concurrencia:** Capacidad de ejecutar múltiples actividades en paralelo o simultáneamente.

-Un **Programa Concurrente** contiene dos o más procesos que trabajan juntos para realizar una tarea.
-Cada proceso es un programa secuencial.
-Un **Programa Secuencial** tiene 1 hilo.
-Un **Programa Concurrente** contiene múltiples hilos de control.
-El orden en el que se ejecutan procesos concurrentes no es determinístico.

Relación entre independencia y cooperante
* Un programa se puede paralelizar si tiene partes independientes.

**Interleaving (Intercalamiento):** Técnica de ejecución donde las instrucciones de distintos procesos se alternan en el tiempo, de modo que en un tiempo determinado solo se está ejecutando una única **acción atómica**. Esta es la única manera en la que un entorno monoprocesador puede lograr que los procesos o hilos trabajen de manera concurrente. El planificador del sistema operativo alterna rápidamente entre las tareas mediante cambios de contexto, dando el aspecto de simultaneidad aunque no exista un paralelismo real.

**Overlaping (Solapamiento):** Consiste en la ejecución simultanea de instrucciones permitiendo que múltiples acciones atómicas se ejecuten en el mismo instante de tiempo. Esta modalidad solo puede obtenerse en entornos multiprocesadores o nucleos múltiples donde los hilos o procesos se ejecutan realmente en paralelo.
El solapamiento está relacionado con la capacidad de paralelizar acciones, lo cual **es posible siempre que los procesos sean independientes,** es decir, que trabajen con distintos recursos y no dependan uno de otros para ser realizados.

**Interleaving:** Intercambio temporal.

**Overlapping:** Ejecución paralela real permitida por hardware.

**Condición de Bernstein:** El conjunto de lectura de un proceso es disjunto del conjunto de lectura y escritura del otro.

La diferencia estructural entre **Procesos y Threads**: Los hilos dentro de un mismo proceso comparten el código, los datos y los archivos, pero cada uno tiene sus propios registros y **stack** (pila). Esto es vital para entender por qué la comunicación entre hilos es más rápida que entre procesos.
